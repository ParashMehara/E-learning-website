Object-Oriented Programming

Object-Oriented Programming (OOP) is a programming paradigm that uses "objects" to model real-world entities. In Java, everything revolves around objects and classes. This section covers the core concepts of OOP and how they are implemented in Java.

1. Classes and Objects

In Java, a class is a blueprint for creating objects. An object is an instance of a class. Classes encapsulate data for the object and methods to manipulate that data.

Defining a Class

Here's an example of how to define a class in Java:

public class Car {
    // Attributes
    String color;
    String model;

    // Constructor
    public Car(String color, String model) {
        this.color = color;
        this.model = model;
    }

    // Method
    public void displayInfo() {
        System.out.println("Model: " + model + ", Color: " + color);
    }
}

Creating an Object

Once you have a class, you can create objects of that class:

public class Main {
    public static void main(String[] args) {
        // Creating an object of the Car class
        Car myCar = new Car("Red", "Toyota");
        myCar.displayInfo();
    }
}

2. Inheritance

Inheritance is a mechanism where a new class derives properties and behavior (methods) from an existing class. The class that is inherited from is called the superclass, and the class that inherits is called the subclass.

Defining a Subclass

Here's an example of how to use inheritance in Java:

public class ElectricCar extends Car {
    // Additional attribute
    int batteryCapacity;

    // Constructor
    public ElectricCar(String color, String model, int batteryCapacity) {
        super(color, model);
        this.batteryCapacity = batteryCapacity;
    }

    // Method
    public void displayBatteryInfo() {
        System.out.println("Battery Capacity: " + batteryCapacity + " kWh");
    }
}

Using the Subclass

Creating an object of the subclass:

public class Main {
    public static void main(String[] args) {
        // Creating an object of the ElectricCar class
        ElectricCar myElectricCar = new ElectricCar("Blue", "Tesla", 75);
        myElectricCar.displayInfo();
        myElectricCar.displayBatteryInfo();
    }
}

3. Polymorphism

Polymorphism allows objects to be treated as instances of their parent class rather than their actual class. It is achieved through method overriding and method overloading.

Method Overriding

Method overriding allows a subclass to provide a specific implementation of a method that is already defined in its superclass:

public class Car {
    public void start() {
        System.out.println("Car is starting");
    }
}

public class ElectricCar extends Car {
    @Override
    public void start() {
        System.out.println("Electric car is starting silently");
    }
}

Method Overloading

Method overloading allows multiple methods with the same name but different parameters within the same class:

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }
}

4. Encapsulation

Encapsulation is the practice of wrapping the data (attributes) and code (methods) together as a single unit. It helps to protect the data by restricting direct access to it and exposing only necessary methods.

Using Encapsulation

Here's an example of encapsulation in Java:

public class Person {
    // Private attributes
    private String name;
    private int age;

    // Public getter and setter methods
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}

5. Conclusion

Object-Oriented Programming is a powerful paradigm that helps in creating organized, modular, and reusable code. By mastering OOP principles like classes, objects, inheritance, polymorphism, and encapsulation, you can develop more robust and maintainable Java applications.
