Exception Handling

Exception handling in C++ provides a way to react to exceptional circumstances (like runtime errors) in programs. It allows you to manage errors gracefully and maintain normal program flow even when an error occurs.

Basics of Exception Handling

C++ uses three main keywords for exception handling:

  • try: Used to define a block of code where exceptions might occur.
  • catch: Used to define a block of code that handles exceptions thrown by the try block.
  • throw: Used to throw an exception when an error condition is detected.

Example of Exception Handling

Here's a simple example that demonstrates how to use these keywords to handle exceptions:

#include <iostream>

void divide(int a, int b) {
    if (b == 0) {
        throw std::runtime_error("Division by zero error");
    }
    std::cout << "Result: " << (a / b) << std::endl;
}

int main() {
    try {
        divide(10, 2); // This will work
        divide(10, 0); // This will throw an exception
    } catch (const std::runtime_error &e) {
        std::cerr << "Caught an exception: " << e.what() << std::endl;
    }

    return 0;
}

Exception Classes

In C++, you can define your own exception classes by inheriting from the std::exception class. This allows you to create custom exceptions with additional information:

#include <iostream>
#include <exception>

class MyException : public std::exception {
private:
    std::string message;
public:
    MyException(const std::string &msg) : message(msg) {}

    virtual const char* what() const noexcept {
        return message.c_str();
    }
};

void riskyOperation() {
    throw MyException("Something went wrong in riskyOperation");
}

int main() {
    try {
        riskyOperation();
    } catch (const MyException &e) {
        std::cerr << "Caught custom exception: " << e.what() << std::endl;
    }

    return 0;
}

Best Practices for Exception Handling

  • Use exceptions for exceptional conditions: Do not use exceptions for routine control flow.
  • Catch exceptions by reference: Catch exceptions by reference to avoid object slicing and unnecessary copies.
  • Provide meaningful exception messages: Include descriptive messages to help diagnose the problem.
  • Ensure exceptions do not leave resources in an invalid state: Use RAII (Resource Acquisition Is Initialization) or smart pointers to manage resources automatically.

Conclusion

Exception handling is a crucial part of robust C++ programming. By properly using try, catch, and throw, along with custom exception classes, you can handle errors effectively and keep your program running smoothly even in the face of unexpected situations.
